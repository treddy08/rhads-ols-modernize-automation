= ocp4_workload_cluster_mta - Install MTA Operator (Cluster-Wide)

== Role Overview

* This role installs the Migration Toolkit for Applications (MTA) operator cluster-wide.
* The operator is installed in the `openshift-mta` namespace using the `install_operator` role pattern.
* This role only installs the operator itself - it does NOT create per-user MTA instances.
* Per-user MTA instances should be created using the `ocp4_workload_mta_tenant` role.

== Architecture

=== Cluster-Wide Operator Installation

This role installs the MTA operator once at the cluster level:

----
OpenShift Cluster
├── openshift-mta namespace
│   └── MTA Operator (watches all namespaces)
└── User namespaces (created by ocp4_workload_mta_tenant)
    ├── user0 → Tackle CR (MTA instance)
    ├── user1 → Tackle CR (MTA instance)
    └── userN → Tackle CR (MTA instance)
----

=== Why Cluster-Wide?

* *Resource Efficiency*: One operator serves all users instead of N operators
* *Simplified Management*: Update once, all users benefit
* *Design Pattern*: MTA operator is designed for cluster-wide installation
* *CRD Management*: Custom Resource Definitions are cluster-scoped

== Installation

The role uses the `install_operator` role to:

1. Create the `openshift-mta` namespace
2. Create an OperatorGroup
3. Create a Subscription to the MTA operator
4. Optionally configure a catalog snapshot

== Removal

The role can remove the MTA operator and namespace by setting `ACTION=destroy`:

1. Removes the CSV (ClusterServiceVersion)
2. Removes the Subscription
3. Removes the InstallPlan
4. Removes the CatalogSource (if configured)
5. Removes the OperatorGroup
6. Removes the `openshift-mta` namespace
7. Cleans up CRD labels and Operator object

== Variables

All variables are defined in `./defaults/main.yml`:

=== Operator Configuration

[cols="2,1,3",options="header"]
|===
| Variable | Default | Description

| `ocp4_workload_cluster_mta_operator_name`
| `mta-operator`
| Name of the operator

| `ocp4_workload_cluster_mta_namespace`
| `openshift-mta`
| Namespace where operator is installed

| `ocp4_workload_cluster_mta_channel`
| `stable-v8.0`
| Operator channel to subscribe to

| `ocp4_workload_cluster_mta_starting_csv`
| `""`
| Optional: Specific CSV version to install

| `ocp4_workload_cluster_mta_automatic_install_plan_approval`
| `true`
| Automatically approve operator updates
|===

=== Catalog Snapshot Configuration

[cols="2,1,3",options="header"]
|===
| Variable | Default | Description

| `ocp4_workload_cluster_mta_catalogsource_setup`
| `false`
| Use a catalog snapshot instead of default catalog

| `ocp4_workload_cluster_mta_catalogsource_image`
| `quay.io/rhpds/olm_snapshot_redhat_catalog`
| Catalog snapshot image

| `ocp4_workload_cluster_mta_catalogsource_image_tag`
| `v4.15_2024_03_11`
| Catalog snapshot image tag
|===

== Usage Example

=== In a Catalog Item

[source,yaml]
----
workloads:
  - rhads_ols_modernize.ocp4_workload_cluster_mta  # Install operator cluster-wide
  - rhads_ols_modernize.ocp4_workload_mta_tenant   # Create per-user instances
----

=== Override Variables

[source,yaml]
----
# Use a specific MTA version
ocp4_workload_cluster_mta_channel: stable-v8.1

# Enable catalog snapshot
ocp4_workload_cluster_mta_catalogsource_setup: true
ocp4_workload_cluster_mta_catalogsource_image_tag: v4.16_2024_06_15
----

== Dependencies

* `install_operator` role from agnosticd-v2
* OpenShift cluster with cluster-admin permissions

== Notes

* This role requires cluster-admin privileges
* The operator will be available cluster-wide after installation
* To create MTA instances for users, use `ocp4_workload_mta_tenant` role
* CRDs (CustomResourceDefinitions) are cluster-scoped and shared across all users
